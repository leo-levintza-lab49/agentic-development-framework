## Phase 1: SVG Error Fixes

### Issue: XML Parsing Errors

Three SVG files contained unescaped ampersand characters (`&`) causing XML parser failures when opened in browsers.

### Root Cause

In XML/SVG, the ampersand character must be encoded as `&amp;`. Using bare `&` creates invalid XML:

```xml
<!-- ❌ Invalid XML -->
<text>Scripts & Templates</text>

<!-- ✅ Valid XML -->
<text>Scripts &amp; Templates</text>
```

**Boundary Checking**:
```python
import re

def check_svg_bounds(filename):
    with open(filename, 'r') as f:
        content = f.read()
    
    issues = []
    
    # Check text near right edge
    text_pattern = r'<text[^>]*x="(\d+)"[^>]*>([^<]+)</text>'
    for match in re.finditer(text_pattern, content):
        x = int(match.group(1))
        if x > 1800:  # Warning threshold
            issues.append(f"Text at x={x} near edge")
    
    # Check boxes extending beyond canvas
    rect_pattern = r'<rect[^>]*x="(\d+)"[^>]*width="(\d+)"'
    for match in re.finditer(rect_pattern, content):
        x = int(match.group(1))
        width = int(match.group(2))
        if x + width > 1920:  # Canvas width
            issues.append(f"Box overflow: {x} + {width} > 1920")
    
    return issues
```

### XML Entity Encoding

**Required Escapes in XML/SVG**:

| Character | Entity | Context |
|-----------|--------|---------|
| `&` | `&amp;` | Always (this session's bug) |
| `<` | `&lt;` | In text content |
| `>` | `&gt;` | In text content |
| `"` | `&quot;` | In attribute values |
| `'` | `&apos;` | In attribute values |

**Common Mistakes**:
```xml
<!-- ❌ Invalid -->
<text>Scripts & Templates</text>
<text>x < 5</text>
<text attr="5 > 3">...</text>

<!-- ✅ Valid -->
<text>Scripts &amp; Templates</text>
<text>x &lt; 5</text>
<text attr="5 &gt; 3">...</text>
```

## Lessons Learned

### SVG Development Best Practices

#### 1. Always Encode Special Characters

**Problem**: Bare ampersands (`&`) cause XML parse errors  
**Solution**: Use entity encoding (`&amp;`)

**Complete encoding table**:
```
& → &amp;   (ampersand)
< → &lt;    (less than)
> → &gt;    (greater than)
" → &quot;  (quotation mark)
' → &apos;  (apostrophe)
```

**Prevention strategy**:
- Use search-and-replace before saving: `& → &amp;`
- Validate with `xmllint` after every edit
- Set up pre-commit hooks to catch encoding errors

#### 2. Respect Canvas Boundaries

**Problem**: Elements positioned too close to edges get clipped  
**Solution**: Maintain minimum margins

**Recommended margins**:
```
Left edge: x ≥ 40px
Right edge: x + width ≤ 1880px (40px margin from 1920)
Top edge: y ≥ 30px
Bottom edge: y + height ≤ 1050px (30px margin from 1080)
```

**For rotated text**: Add extra margin (50-80px) as rotation extends bounds

**Automated checking**:
```python
def validate_bounds(x, width, canvas_width=1920):
    if x < 40:
        return f"Too close to left edge: x={x}"
    if x + width > canvas_width - 40:
        return f"Too close to right edge: x={x}, width={width}"
    return "OK"
```

#### 3. Use Consistent Spacing Formulas

**Problem**: Manual positioning leads to misalignment  
**Solution**: Define base positions and calculate offsets

**Example spacing system**:
```
BASE_Y = 140  # Layer start
ICON_OFFSET = 45
TITLE_OFFSET = 40
SUBTITLE_OFFSET = 65
DETAIL_OFFSET = 83

# Calculate positions
icon_y = BASE_Y + ICON_OFFSET      # 185
title_y = BASE_Y + TITLE_OFFSET    # 180
subtitle_y = BASE_Y + SUBTITLE_OFFSET  # 205
```

**Benefits**:
- Predictable alignment
- Easy to adjust entire layers
- Consistent visual rhythm
- Easier debugging

#### 4. Validate Early and Often

**Problem**: Accumulated errors hard to debug  
**Solution**: Validate after each significant change

**Validation workflow**:
```bash
# 1. Syntax check
xmllint --noout file.svg

# 2. Visual check in browser
open -a "Google Chrome" file.svg

# 3. Boundary check
python check_bounds.py file.svg

# 4. PowerPoint import test (periodically)
# Import into PowerPoint and check rendering
```

**Frequency**:
- After every 50-100 lines of code
- Before committing changes
- After any copy-paste operations
- When changing dimensions or positions

#### 5. Structure Before Content

**Problem**: Adding content to poorly structured layout causes cascading issues  
**Solution**: Design the layout structure first

**Layout-first workflow**:
```
1. Define canvas and margins
2. Calculate major sections (header, body, footer)
3. Define grid/column system
4. Add placeholder boxes at calculated positions
5. Validate structure (check spacing, alignment)
6. Add content to boxes
7. Final validation
```

**Template approach**:
```xml
<!-- Structure template -->
<svg viewBox="0 0 1920 1080">
  <!-- Section 1: Header (y=0-100) -->
  <rect x="40" y="30" width="1840" height="70" fill="red" opacity="0.3"/>
  
  <!-- Section 2: Content (y=120-900) -->
  <rect x="40" y="120" width="1840" height="780" fill="blue" opacity="0.3"/>
  
  <!-- Section 3: Footer (y=920-1050) -->
  <rect x="40" y="920" width="1840" height="130" fill="green" opacity="0.3"/>
</svg>
```

Once structure validates, replace colored boxes with actual content.

### Alignment Techniques

#### 1. Grid-Based Layout

**Principle**: Define a grid system and snap elements to grid lines

**12-column grid example for 1920px width**:
```
Usable width: 1920 - 80 (margins) = 1840px
Column width: 1840 / 12 = 153.33px
Gutter: 20px

Column positions:
Col 1: x = 40
Col 2: x = 40 + 153.33 + 20 = 213.33
Col 3: x = 386.66
...
```

**Snap elements to columns**:
```xml
<!-- Full width card (12 columns) -->
<rect x="40" y="100" width="1840" height="200"/>

<!-- Half width cards (6 columns each) -->
<rect x="40" y="100" width="900" height="200"/>
<rect x="980" y="100" width="900" height="200"/>

<!-- Third width cards (4 columns each) -->
<rect x="40" y="100" width="586" height="200"/>
<rect x="666" y="100" width="586" height="200"/>
<rect x="1293" y="100" width="586" height="200"/>
```

#### 2. Vertical Rhythm

**Principle**: Use consistent vertical spacing (baseline grid)

**8px baseline example**:
```
All vertical positions = 8 * N

Base unit: 8px
Spacing options: 8, 16, 24, 32, 40, 48...

Example:
Title: y = 40 (8 * 5)
Subtitle: y = 64 (8 * 8) → 24px gap
Body: y = 88 (8 * 11) → 24px gap
```

**Benefits**:
- Harmonious visual spacing
- Easier calculations
- Scalable system

#### 3. Text Baseline Alignment

**Problem**: Text with different font sizes doesn't align visually  
**Solution**: Understand baseline positioning

**Font metrics**:
```
Font size: 20px
Baseline: ~80% from top (16px down from y position)
Cap height: ~70% of font size
Descenders: extend below baseline

For y=100, font-size=20:
  Top of text: y=100
  Baseline: y=116 (100 + 20*0.8)
  Bottom: y=120
```

**Aligning different sizes**:
```xml
<!-- Baseline alignment for different font sizes -->
<!-- Target: all text baseline at y=200 -->

<!-- 24px font: y = 200 - (24 * 0.8) = 200 - 19.2 = 180.8 -->
<text x="100" y="181" font-size="24">Large</text>

<!-- 16px font: y = 200 - (16 * 0.8) = 200 - 12.8 = 187.2 -->
<text x="200" y="187" font-size="16">Medium</text>

<!-- 12px font: y = 200 - (12 * 0.8) = 200 - 9.6 = 190.4 -->
<text x="300" y="190" font-size="12">Small</text>
```

#### 4. Icon-Text Pairing

**Problem**: Emoji icons and text don't align naturally  
**Solution**: Offset emoji by ~60% of font size

**Formula**:
```
For icon font-size=40, text font-size=16:
  
Text y-position (baseline): 200
Icon offset: 40 * 0.6 = 24
Icon y-position: 200 - 24 = 176

Result: Icon appears vertically centered with text
```

**Implementation**:
```xml
<!-- Text at y=200 -->
<text x="100" y="200" font-size="16">Label</text>

<!-- Icon offset up by 60% of its size -->
<text x="50" y="176" font-size="40">📚</text>
```

#### 5. Boundary Verification

**Automated checking script**:
```python
def verify_element_bounds(x, y, width, height, canvas_w=1920, canvas_h=1080, margin=40):
    """
    Verify element fits within canvas with margins
    """
    errors = []
    
    # Check left edge
    if x < margin:
        errors.append(f"Left overflow: x={x} < margin={margin}")
    
    # Check right edge
    if x + width > canvas_w - margin:
        errors.append(f"Right overflow: x={x} + width={width} > {canvas_w - margin}")
    
    # Check top edge
    if y < margin:
        errors.append(f"Top overflow: y={y} < margin={margin}")
    
    # Check bottom edge
    if y + height > canvas_h - margin:
        errors.append(f"Bottom overflow: y={y} + height={height} > {canvas_h - margin}")
    
    return errors

# Usage
errors = verify_element_bounds(x=1800, y=100, width=200, height=100)
# Returns: ['Right overflow: x=1800 + width=200 > 1880']
```

**Goal**: Prevent code and documentation from drifting out of sync

**Triggers**:
1. **SessionStart hook**: Warns about stale or missing documentation
2. **SessionStop hook**: Blocks session exit if documentation not updated

#### Hook Logic

**SessionStart**:
```python
if last_code_change > last_doc_change + 21_days:
    warn("Documentation may be stale")

if not exists("ARCHITECTURE.md") or not exists("SETUP.md"):
    warn("Missing recommended docs")
```

**SessionStop**:
```python
if has_significant_changes and (docs_missing or docs_stale):
    if docs_missing:
        suggest("/doc-generate")
        return "block"
    else:
        suggest("/doc-update")
        return "block"
else:
    return "approve"
```

**Counts as significant**:
- API changes (function signatures, endpoints)
- Architecture changes (new services, data flow)
- Configuration changes (new settings, environment variables)
- New features (user-facing functionality)
- Build/deployment changes (scripts, CI/CD)

**Does NOT count as significant**:
- Bug fixes (internal logic only)
- Refactoring (no behavior change)
- Test additions (unless testing strategy changes)
- Comment updates
- Formatting changes
