### Analysis Scripts
```python
# Python script to check SVG boundaries
import re

def check_svg_bounds(filename):
    with open(filename, 'r') as f:
        content = f.read()
    
    issues = []
    
    # Check for text near right edge (x > 1800)
    text_pattern = r'<text[^>]*x="(\d+)"[^>]*>([^<]+)</text>'
    for match in re.finditer(text_pattern, content):
        x = int(match.group(1))
        text = match.group(2)
        if x > 1800:
            issues.append(f"Text near right edge at x={x}: '{text[:50]}...'")
    
    # Check for boxes extending beyond canvas (x + width > 1920)
    rect_pattern = r'<rect[^>]*x="(\d+)"[^>]*width="(\d+)"'
    for match in re.finditer(rect_pattern, content):
        x = int(match.group(1))
        width = int(match.group(2))
        if x + width > 1920:
            issues.append(f"Box extends beyond canvas: x={x}, width={width}")
    
    return issues
```

## Lessons Learned

### SVG Best Practices
1. **Always escape special characters**: `&` → `&amp;`, `<` → `&lt;`, `>` → `&gt;`
2. **Leave margin from canvas edges**: Minimum 30-40px from boundaries
3. **Use consistent spacing formulas**: Define base positions and offsets
4. **Test with validation tools**: `xmllint` catches syntax errors early
5. **Visual hierarchy matters**: Larger cards need more internal spacing

### Alignment Techniques
1. **Grid-based layout**: Calculate positions with consistent gaps
2. **Vertical rhythm**: Use multiples of base spacing (e.g., 5px increments)
3. **Text baseline alignment**: Match font sizes with y-position offsets
4. **Icon-text pairing**: Offset emoji baseline by ~60% of font size
5. **Boundary calculations**: Verify `x + width ≤ canvas_width - margin`

### PowerPoint Compatibility
1. **Simple SVG structure**: Avoid complex filters or gradients
2. **Standard fonts**: Use system font stacks for reliability
3. **Proper viewBox**: Match viewBox to canvas dimensions (0 0 1920 1080)
4. **No external dependencies**: Embed all styles and markers
