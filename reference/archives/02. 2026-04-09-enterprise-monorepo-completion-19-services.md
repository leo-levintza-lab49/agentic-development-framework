## Known Issues

### Naming Convention Bug

The Java service generator has a class name conversion issue:
- Generates: `UuserUservice` instead of `UserService`
- Generates: `UpaymentUservice` instead of `PaymentService`
- Location: Line 107 in `generate-monorepo-service.sh`

**Current Code:**
```bash
local class_prefix=$(echo "$service_name" | sed -r 's/(^|-)([a-z])/\U\2/g')
```

**Impact:**
- Services work correctly
- Code has poor naming conventions
- Should be fixed in future iteration

**Workaround:** Services are functional despite naming issues.
